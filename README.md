# StudentProject

ASP.NET Core REST API для управления событиями.

## Сборка и запуск

1. Откройте терминал в корне проекта.
2. Соберите проект: dotnet build
3. Запустите проект: dotnet run
4. После запуска API будет доступен по адресу, указанному в консоли (например, https://localhost:5001/).

## Тестирование

1. Откройте терминал в корне проекта.
2. Тестирование: dotnet test. Эта команда запустит все тесты, определённые в проекте, и выведет результаты в консоль.

## Обработка исключений

Для глобальной обработки исключений используется формат Problem Details (RFC 7807)

## Swagger

Для быстрого тестирования и знакомства с API используйте Swagger UI по адресу: https://localhost:PORT/swagger
*(PORT – фактический порт, выбранный при запуске; смотрите вывод запуска приложения)*

## Реализованные эндпоинты

### Получение всех событий GET /events с учетом фильтров:  
1. часть имени
2. дата начала
3. дата конца
4. номер страницы для отображения
5. число элементов на странице
### Получение события по id GET /events/{id}
### Создание нового события POST /events
### Обновление события PUT /events/{id}
### Удаление события DELETE /events/{id}
### Создание брони по id события POST /events/{id}/book
### Получение брони по id GET /bookings/{id}

## Модель Booking (бронь)
Состав полей:
- Id (Guid, обязательное) — уникальный идентификатор брони;
- EventId (Guid, обязательное) — идентификатор события, к которому относится бронь;
- Status (BookingStatus, обязательное) — текущий статус брони:
  - Pending — бронь создана, ожидает обработки;
  - Confirmed — бронь подтверждена;
  - Rejected — бронь отклонена.
- CreatedAt (DateTime, обязательное) — дата и время создания брони;
- ProcessedAt (DateTime?, опциональное) — дата и время обработки брони.

## Фоновая обработка бронирования
- сервис периодически опрашивает хранилище на наличие бронирований в статусе Pending;
- для каждой необработанной брони выполняется искусственная задержка (Task.Delay на 2 секунды), имитирующая обращение к внешней системе;
- после задержки бронь переводится в статус Confirmed;
- после смены статуса заполняется поле ProcessedAt;
- обновлённая бронь сохраняется в хранилище.

## Пример создания брони
- создайте событие через POST /events;
- создайте бронь через POST /events/{id}/book — убедитесь, что получили 202 Accepted и Location;
- сразу запросите GET /bookings/{id} — статус должен быть Pending;
- подождите несколько секунд и повторите запрос — статус должен измениться на Confirmed.
